## 1. 物理层

### 1.1 核心定理

| 定理       | 公式                                        | 适用条件       |
| -------- | ----------------------------------------- | ---------- |
| **奈奎斯特** | $$C\_{max} = 2W \log\_2 V$$               | 无噪声、理想低通信道 |
| **香农**   | $$C\_{max} = W \log\_2(1 + \frac{S}{N})$$ | 有高斯噪声、带宽受限 |

> $$ \frac{S}{N} = 10^{\frac{dB}{10}} $$

```mermaid

    A[信道带宽 W] --> B[奈奎斯特: 2W 波特率上限]
    A --> C[香农: 容量取决于信噪比]
    D[码元种类 V] --> B
    E[信噪比 S/N] --> C
```

## 2. 数据链路层

### 2.1 成帧与差错控制

**成帧方法：**

- 比特填充：`01111110` 标志，遇5个连1插0
- 字符填充：DLE STX / DLE ETX

**CRC 校验流程：**

```mermaid
flowchart LR
    A[数据比特串] --> B[模2除法除以生成多项式 Gx]
    B --> C[余数为 FCS]
    C --> D[附加到数据后发送]
    E[接收方同样除法] --> F{余数=0?}
    F -->|是| G[无差错]
    F -->|否| H[差错，丢弃]
```

**海明码：** $2^r \ge m + r + 1$（r校验位，m数据位）

### 2.2 流量控制与可靠传输

| 协议       | 发送窗口            | 接收窗口            | 重传方式      |
| -------- | --------------- | --------------- | --------- |
| 停止-等待    | 1               | 1               | 超时重传      |
| 后退N帧 GBN | $$2^n -1$$      | 1               | 累积确认，回退N帧 |
| 选择重传 SR  | $$\le 2^{n-1}$$ | $$\le 2^{n-1}$$ | 只重传出错帧    |

信道利用率（停止-等待）：\
$$U = \frac{发送时延}{发送时延 + 2 \times 传播时延}$$

### 2.3 介质访问控制 CSMA/CD

```mermaid
sequenceDiagram
    participant A as 站点A
    participant B as 站点B
    A->>A: 监听信道空闲?
    A->>B: 发送数据
    B->>B: 同时发送，冲突发生
    B->>A: 冲突信号
    A->>A: 停止发送，等待随机退避时间
    Note over A,B: 争用期 = 2τ，最小帧长 = 争用期 × 数据率
```

**截断二进制指数退避：**\
第 k 次重传，等待时间 $r \times 2\tau$，r∈\[0, 2^k-1]，k ≤ 10。

### 2.4 以太网与交换机

- **MAC 地址**：48 位，前 24 位 OUI
- **帧格式**：前导码 8B | 目的 MAC 6B | 源 MAC 6B | 类型 2B | 数据 46\~1500B | FCS 4B
- **交换机**：自学习 MAC 表，隔离冲突域，支持全双工

## 3. 网络层

### 3.1 IP 数据报格式

```mermaid
flowchart LR
    subgraph IP首部[IP 首部 20~60 字节]
        direction LR
        A(版本 4bit) --> B(首部长度 4bit) --> C(区分服务 8bit) --> D(总长度 16bit) --> E(标识 16bit) --> F(标志 3bit) --> G(片偏移 13bit) --> H(TTL 8bit) --> I(协议 8bit) --> J(首部校验和 16bit) --> K(源IP 32bit) --> L(目的IP 32bit)
    end
```

**分片相关**：片偏移以 8 字节为单位，MF=1 表示还有分片，DF=1 禁止分片。

### 3.2 子网划分与 CIDR

- 子网掩码：网络位全 1，主机位全 0
- 子网数 = $2^{借位数}$ （注意全0全1子网）
- 可用主机数 = $2^{主机位} - 2$

**CIDR 无分类编址**：使用 `IP/前缀`，如 `192.168.1.0/24`，支持路由聚合。

### 3.3 核心协议

**ARP**：IP → MAC 映射，广播请求、单播响应。

**ICMP**：差错报告（终点不可达3、超时11）、询问（回显8/0 → Ping）。

**路由协议对比**：

| 协议   | 类型      | 算法           | 传输层     | 特点         |
| ---- | ------- | ------------ | ------- | ---------- |
| RIP  | IGP, DV | Bellman-Ford | UDP 520 | 15跳限制，慢收敛  |
| OSPF | IGP, LS | Dijkstra SPF | IP 89   | 分层区域，快速收敛  |
| BGP  | EGP     | 路径向量         | TCP 179 | AS间路由，策略控制 |

### 3.4 NAT 与 IPv6

- **NAT**：私有地址 ↔ 公共地址转换，NAPT 使用端口号区分
- **IPv6**：128 位地址，简化首部，无分片（PMTU），无校验和

## 4. 传输层

### 4.1 UDP 与 TCP 对比

| 特性      | UDP            | TCP       |
| ------- | -------------- | --------- |
| 连接      | 无连接            | 面向连接      |
| 可靠性     | 不可靠            | 可靠        |
| 首部长度    | 8 字节           | 20\~60 字节 |
| 流量/拥塞控制 | 无              | 有         |
| 适用场景    | DNS, DHCP, 流媒体 | 文件传输, 网页  |

### 4.2 TCP 三次握手与四次挥手

```mermaid
sequenceDiagram
    participant C as 客户端
    participant S as 服务器
    Note over C,S: 三次握手
    C->>S: SYN=1, seq=x
    S->>C: SYN=1, ACK=1, seq=y, ack=x+1
    C->>S: ACK=1, seq=x+1, ack=y+1
    Note over C,S: 连接建立 (ESTABLISHED)
    
    Note over C,S: 四次挥手
    C->>S: FIN=1, seq=u
    S->>C: ACK=1, ack=u+1
    S->>C: FIN=1, ACK=1, seq=v, ack=u+1
    C->>S: ACK=1, seq=u+1, ack=v+1
    Note over C: TIME-WAIT (2MSL)
```

**为什么三次握手？** 防止已失效的连接请求到达服务器。

### 4.3 TCP 流量控制与拥塞控制

**流量控制**：滑动窗口机制，接收方窗口字段控制发送速率。

**拥塞控制**：

```mermaid
graph LR
    A[慢开始] -->|cwnd指数增长| B{达到ssthresh?}
    B -->|是| C[拥塞避免: 线性增长]
    C -->|超时| D[ssthresh=cwnd/2, cwnd=1, 重回慢开始]
    C -->|3个重复ACK| E[快重传+快恢复]
    E --> F[ssthresh=cwnd/2, cwnd=ssthresh, 进入拥塞避免]
```

## 5. 应用层

### 5.1 DNS 域名解析

```mermaid
flowchart TD
    A[用户输入域名] --> B{本地缓存?}
    B -->|是| C[返回IP]
    B -->|否| D[本地DNS服务器]
    D --> E[根域名服务器]
    E --> F[.com顶级域名服务器]
    F --> G[权威域名服务器]
    G --> H[返回IP记录]
    H --> C
```

**记录类型**：A (IPv4)、AAAA (IPv6)、CNAME (别名)、MX (邮件)、NS (域名服务器)

### 5.2 HTTP 协议

- **请求报文**：方法 (GET/POST) | URL | 版本 | 头部字段
- **响应报文**：状态码 (2xx成功, 3xx重定向, 4xx客户端错误, 5xx服务器错误)
- **HTTP/2**：二进制分帧、多路复用、头部压缩

### 5.3 其他应用协议

| 协议   | 端口            | 传输层 | 用途          |
| ---- | ------------- | --- | ----------- |
| FTP  | 21(控制)/20(数据) | TCP | 文件传输        |
| SMTP | 25            | TCP | 发送邮件        |
| POP3 | 110           | TCP | 接收邮件(下载删除)  |
| IMAP | 143           | TCP | 接收邮件(服务器保留) |
| DHCP | 67/68         | UDP | 动态IP配置      |

**DHCP 交互过程**：Discover → Offer → Request → ACK
