──[ 1.0 ]──────────────────────────────────────────────────────────[ 文件管理 ]

本章规定文章文件的命名、存放和资源管理规范。

┌─ 1.1 ───────────────────────────────────────────────────[ 文件命名 ]─┐

    文件名格式：

      Topic-Name.md

    规则：
    - 使用英文，单词以连字符 "-" 连接
    - 首字母大写，每个独立单词首字母大写（PascalCase）
    - 不得包含空格、下划线、中文或特殊字符
    - 扩展名必须为 .md

    正确示例：
      NETGEAR-EXS27-0.md
      Pwn-Trick-Notes.md
      Xpdf-CVE-2019-13288.md

    错误示例：
      netgear_exs27.md          （下划线）
      pwn trick notes.md         （空格）
      固件分析.md                （中文文件名）
      Article!.md                （特殊字符）

┌─ 1.2 ───────────────────────────────────────────────────[ 存放位置 ]─┐

    所有文章存放在 src/content/philes/ 下，按主题分卷：

      src/content/philes/
      ├─ volume-0/          ← 本站说明、规则手册
      ├─ volume-1/          ← 技术文章
      │   ├─ pwn-notes/     ← PWN 相关
      │   ├─ fuzz/          ← Fuzz 相关
      │   ├─ projects/      ← 项目类
      │   └─ write-ups/     ← CTF Write-up
      └─ ...

    子目录命名规则与文件名相同：英文、连字符、无空格。

┌─ 1.3 ───────────────────────────────────────────────────[ 图片资源 ]─┐

    图片源文件存放在 public/images/ 目录下，按卷和文章子目录组织：

      public/images/
      └─ volume-1/
          └─ fuzz/
              ├─ article-slug/
              │   ├─ screenshot-01.png
              │   └─ diagram.svg
              └─ ...

    图片通过完整路径 Blog\public\images\ 直接提供访问，无需构建时复制。

    **推荐：完整路径**

    使用包含 Blog\public\images\ 根目录的完整路径引用图片：

      ![截图说明](Blog\public\images\volume-1\article-slug\screenshot-01.png)

    路径格式：Blog\public\images\[volume]\[article-slug]\[image-filename]

    **兼容：外部 URL**

      ![外部图片](https://example.com/image.png)

    支持格式：PNG、JPG、WebP、SVG、GIF。

──[ 2.0 ]────────────────────────────────────────────────────────[ 元数据配置 ]

每篇文章开头必须有 YAML frontmatter 元数据块，夹在 --- 之间。

┌─ 2.1 ───────────────────────────────────────────[ Frontmatter 字段 ]─┐

    ```yaml
    ---
    title: "文章标题"          # 必需，支持中文
    date: 2026-06-19           # 必需，YYYY-MM-DD 格式
    author: "作者名"           # 必需
    lang: zh                   # 可选，语言 (en/zh)，默认 en
    order: 0                   # 可选，数字越小越靠前，默认 0
    redacted: false            # 可选，true 则隐藏文章，默认 false
    ---
    ```

    字段说明：
    - title   ：显示在文章顶部和浏览器标签页
    - date    ：发布日期，决定文章在卷内的排序
    - author  ：作者署名，显示在标题下方
    - lang    ：文章语言，zh 为中文，en 为英文
    - order   ：手动排序权重，覆盖日期排序
    - redacted：设为 true 时文章不会出现在目录中

┌─ 2.2 ───────────────────────────────────────────────[ 排序与隐藏 ]─┐

    排序优先级：
    1. order 字段（小 → 大）
    2. date 字段（新 → 旧）

    隐藏文章（不想公开但保留文件）：
      redacted: true

──[ 3.0 ]──────────────────────────────────────────────────────────[ 文章结构 ]

本章规定文章的整体布局和各个区块的书写规范。

┌─ 3.1 ───────────────────────────────────────────[ 目录（CONTENTS） ]─┐

    在 frontmatter 之后、正文之前，可以有一个 CONTENTS 块，列出文章
    所有章节的编号和标题。CONTENTS 内容以标准 Markdown 格式渲染，
    你可以自由使用 Markdown 语法进行排版。

    示例：

      CONTENTS

      1.0 Summary
      1.1 Motivation
      2.0 Analysis
      3.0 References

    规则：
    - 以单独一行的 CONTENTS 开头（大小写不敏感）
    - CONTENTS 下方的内容作为标准 Markdown 文本渲染
    - 可以使用 Markdown 列表、缩进、表格等任意语法排版
    - 块以空行结束

    推荐用法：
    - 使用无序列表（`- 1.0  Title`）实现层级缩进
    - 或使用纯文本 + 空格缩进实现自定义排版

    示例（Markdown 无序列表）：

      CONTENTS

      - 1.0  Summary
        - 1.1  Motivation
      - 2.0  Analysis
      - 3.0  References

    示例（纯文本缩进）：

      CONTENTS

      1.0  Summary
        1.1  Motivation
      2.0  Analysis
      3.0  References

┌─ 3.2 ─────────────────────────────────────────────────[ 章节标题 ]─┐

    正文中使用虚线框标题分隔章节：

      ──[ 1.0 ]──...──[ Summary ]

    格式：
    - 以 ──[ 开头，编号居中，]──...──[ 标题 ]  结尾
    - 编号格式：数字 . 数字（如 1.0、2.1、3.2）
    - 标题前后各有一个空格
    - 标题上下各空一行

    约定：
    - 0.0 ：摘要/概述（Summary）
    - 1.0 ：第一章节
    - 2.0 ：第二章节
    - 子章节使用 2.1、2.2 等

┌─ 3.3 ───────────────────────────────────────────────────[ 摘要区 ]─┐

    摘要区位于文章第一个章节（通常为 1.0 Summary），包含以下元素：

    (1) 引用块（可选）
      使用 > 前缀列出关键信息：

        > target      : 目标设备/软件
        > result      : 分析结果

    (2) 图例说明（INK KEY）
      使用树形结构列出颜色含义（见 3.4 节）

    (3) 正文概述
      1-3 段简要说明文章内容

┌─ 3.4 ─────────────────────────────────────────────────[ 图例说明 ]─┐

    在文章开头定义颜色图例，帮助读者理解后续 ANSI 颜色标记的含义。

    格式：

        INK KEY:
        ├─ #[C|C]  format markers and container nodes
        ├─ #[G|G]  validated or recovered data
        ├─ #[Y|Y]  offsets, sizes, boundary values
        └─ #[R|R]  wrong paths, hazards, negative controls

    规则：
    - 首行：两个空格 + LABEL:（如 INK KEY:）
    - 分支：两个空格 + ├─ 或 └─ + 空格 + ANSI 颜色标记 + 说明
    - 每个分支对应一种颜色含义

┌─ 3.5 ───────────────────────────────────────────────────[ 流程图 ]─┐

    使用框线字符和箭头绘制数据流向或流程结构。

    格式：

        FLOW LABEL:
        root node
            │
            ▼
        child node
            ├─ offset  #[C|description]
            ├─ offset  #[G|description]
            └─ offset  #[Y|description]

    规则：
    - 首行：两个空格 + 标签 + 冒号
    - 节点：缩进 4 个空格
    - 连线：│ 缩进 4 个空格
    - 箭头：▼ 缩进 4 个空格
    - 分支：├─ 或 └─ 缩进 4 个空格
    - 关键数据使用 ANSI 颜色标记

┌─ 3.6 ───────────────────────────────────────────────────[ 证明链 ]─┐

    用于展示验证步骤的有序列表，每步包含检查项和验证结果。

    格式：

        PROOF CHAIN:
        ├─ [1] #[C|wrapper header]
        │  signal : description
        ├─ [2] #[G|positive control]
        │  signal : description
        └─ [3] #[R|negative control]
           signal : description

    规则：
    - 首行：两个空格 + 标签 + 冒号
    - 每步：两个空格 + ├─ 或 └─ + [序号] + ANSI 颜色标记
    - 信号行：两个空格 + │ + 两个空格 + signal : 描述
    - 最后一步：使用 └─，信号行缩进 3 个空格（无 │）

┌─ 3.7 ─────────────────────────────────────────────────────[ 表格 ]─┐

    使用框线字符绘制表格，所有内容手动对齐。

    格式：

        ┌──────────────────┬──────────────┬──────────────────┐
        │ Field            │ Offset       │ Value            │
        ├──────────────────┼──────────────┼──────────────────┤
        │ magic            │ 0x00000000   │ #[G|ENCR]        │
        │ version          │ 0x00000004   │ #[C|0x00000001]  │
        └──────────────────┴──────────────┴──────────────────┘

    规则：
    - 表格缩进 2 个空格
    - 顶框：┌─┬─┐，中框：├─┼─┤，底框：└─┴─┘
    - 表头行和数据行均用 │ 分隔
    - 每列手动对齐，空格填充
    - 关键数据使用 ANSI 颜色标记

──[ 4.0 ]──────────────────────────────────────────────────────────[ 文本格式 ]

本章规定正文文本的 Markdown 格式规范。

┌─ 4.1 ───────────────────────────────────────────────[ 段落与换行 ]─┐

    段落之间用空行分隔。连续两行之间如果没有空行，会被合并为同一段落。

    正确：
      这是第一段。

      这是第二段。

    错误：
      这是第一段。
      这还是第一段，因为没有空行分隔。

    引用块使用 > 前缀：

      > 这是引用内容
      > 可以多行

┌─ 4.2 ───────────────────────────────────────────────[ 标题层级 ]─┐

    文章正文中的 Markdown 标题会自动转换为 ANSI 风格：

      # 一级标题     → 粗体分隔线 + 亮黄色文字
      ## 二级标题    → ~ 亮黄色文字 + 下划线
      ### 三级标题   → 黄色文字（缩进）
      #### 四级标题  → 青色文字（缩进）

    注意：一级标题（#）会生成贯穿整行的分隔线，适合用于文章大标题。
    章节分隔请使用──[ N ]──...──[ Title ]格式（见 3.2 节）。

┌─ 4.3 ───────────────────────────────────────────────[ 行内格式 ]─┐

    支持以下行内 Markdown 格式：

      **粗体文本**      → 亮白色强调
      *斜体文本*        → 青色强调
      `行内代码`        → 亮绿色等宽字体
      ~~删除线~~        → 紫色删除线

    注意：行内代码 ` ` 中的 | 字符会被自动保护，不会干扰表格解析。
    但请避免在行内代码中使用复杂的 ANSI 标记。

┌─ 4.4 ─────────────────────────────────────────────────────[ 链接 ]─┐

    标准 Markdown 链接语法：

      [链接文字](https://example.com)

    渲染为亮青色可点击链接。链接文字应与目标页面内容相关，避免使用
    "点击这里" 等无意义文字。

    内部引用（如引用同一文章中的公式）使用锚点链接：

      [公式 (3)](#eq-entropy)

──[ 5.0 ]────────────────────────────────────────────────────[ 代码块与容器 ]

本章规定代码块和提示容器的书写规范。

┌─ 5.1 ───────────────────────────────────────────[ 围栏代码块 ]─┐

    使用三个反引号包裹代码，并指定语言标识：

      ```bash
      echo "hello world"
      chmod +x script.sh
      ```

    渲染效果：上下边框 + 语言标签，代码内容自动语法高亮。

    支持的语言标识（常用）：
    - bash / shell ：Shell 脚本
    - python / py  ：Python
    - c / cpp      ：C / C++
    - javascript   ：JavaScript
    - rust         ：Rust
    - 以及 highlight.js 支持的所有语言

    代码块中的内容不会被 ANSI 颜色标记解析，可以安全地包含 #、[、] 等
    特殊字符。

┌─ 5.2 ───────────────────────────────────────────────[ ::: 容器块 ]─┐

    使用 ::: 标记创建带边框的提示容器，与代码块视觉效果明显区分。

    基本语法：

      :::important
      这是重要提示内容
      :::

    支持带标题的格式：

      :::warning[安全警告]
      请勿在生产环境执行此操作
      :::

    或空格分隔：

      :::note 补充说明
      这是补充信息
      :::

    容器类型与颜色：

      important  → 亮黄色边框，用于重要提示
      note       → 青色边框，用于备注信息
      tip        → 绿色边框，用于技巧提示
      warning    → 红色边框，用于警告信息

    规则：
    - 容器内支持 Markdown 格式和 ANSI 颜色标记
    - 容器内也支持代码块
    - 容器不可以嵌套

──[ 6.0 ]────────────────────────────────────────────────────────[ 数学公式 ]

使用 KaTeX 渲染 LaTeX 数学公式，支持行内和块级两种模式。

┌─ 6.1 ───────────────────────────────────────────────[ 行内公式 ]─┐

    使用单个 $ 包裹：

      $E = mc^2$

    行内公式嵌入在段落文字中，不单独成行。适用于简短表达式。

    自动提升规则：当行内公式包含以下结构时，会自动提升为块级公式：
    - 多行结构（cases, aligned, array, matrix 等）
    - 运算符 + 分数（如 \lim_{x \to 0} \frac{1}{x}）
    - 嵌套分数（\frac 内部再包含 \frac）

┌─ 6.2 ───────────────────────────────────────────────[ 块级公式 ]─┐

    使用两个 $$ 包裹：

      $$
      S = k \log W
      $$

    块级公式独占一行，居中对齐，自动编号。适用于重要公式或复杂表达式。

    公式过长时，会自动出现水平滚动条，不影响页面布局。

┌─ 6.3 ───────────────────────────────────────[ 公式编号与引用 ]─┐

    在公式后添加标签，可在正文中引用：

      $$
      S = k \log W \label{eq-entropy}
      $$

    正文中引用：

      如公式 \ref{eq-entropy} 所示，熵与微观状态数的对数成正比。

    引用渲染为可点击的链接，点击后跳转到对应公式。

──[ 7.0 ]──────────────────────────────────────────────────────────────[ 图片 ]

┌─ 7.1 ───────────────────────────────────────────────[ 图片插入 ]─┐

    图片必须独占一行，使用 Markdown 图片语法：

      ![图片描述](Blog\public\images\volume-1\article-slug\image.png)

    或 HTML 语法：

      <img src="Blog\public\images\volume-1\article-slug\image.png" alt="图片描述">

    规则：
    - 图片行前后必须有空行（即图片独占一个段落）
    - alt 文本（图片描述）必须填写，用于 SEO 和无障碍访问
    - 图片统一使用完整路径，格式：`Blog\public\images\[volume]\[article-slug]\[image-filename]`
    - 图片源文件直接存放在 public/images/ 目录下，无需构建时复制
    - 图片会自动添加懒加载（lazy loading）

┌─ 7.2 ───────────────────────────────────────────────[ 图片查看器 ]─┐

    点击图片会弹出全屏查看器，支持以下功能：
    - 鼠标滚轮缩放（每次 10%）
    - 按钮缩放（+/- 按钮，每次 25%）
    - 拖拽平移（放大后可使用鼠标拖拽）
    - 键盘操作：← → 切换图片，+/- 缩放，0 还原，Esc 关闭
    - 移动端支持双指缩放
    - 多张图片时自动显示上一张/下一张按钮

    缩放范围：12% ~ 800%

──[ 8.0 ]──────────────────────────────────────────────[ ANSI 颜色与特殊效果 ]

本章规定 ANSI 终端风格颜色标记和特殊效果的使用规范。

┌─ 8.1 ───────────────────────────────────────[ ANSI 颜色标记 ]─┐

    格式：#[颜色代码|文字内容]

    标准颜色（小写字母）：

      #[r|红色]   #[g|绿色]   #[y|黄色]   #[b|蓝色]
      #[m|紫色]   #[c|青色]   #[w|白色]   #[k|黑色]

    亮色（大写字母）：

      #[R|亮红]   #[G|亮绿]   #[Y|亮黄]   #[B|亮蓝]
      #[M|亮紫]   #[C|亮青]   #[W|亮白]   #[K|灰色]

    功能说明：
    - 红色系（r/R）：错误、警告、负面信息
    - 绿色系（g/G）：成功、验证通过、正确数据
    - 黄色系（y/Y）：注意、偏移量、边界值、重要提示
    - 蓝色系（b/B）：次要信息、强调
    - 青色系（c/C）：格式标记、容器节点、链接
    - 紫色系（m/M）：特殊标记、删除线
    - 白色系（w/W）：默认文本、粗体强调

    重要规则：
    - ANSI 标记不支持嵌套
    - 需要显示 #、[、]、|、\ 时使用反斜杠转义：\[、\]、\|、\\
    - 颜色标记仅用于强调关键信息，不要滥用

┌─ 8.2 ───────────────────────────────────────[ INK 渐变文字 ]─┐

    INK 块用于实现逐字不同颜色的渐变效果。

    语法：
      --[ ink ]--
      |文字行内容
      ~颜色掩码

    规则：
    - 以 --[ ink ]-- 开头，空行结束
    - | 开头：要显示的文字行，| 不会渲染
    - ~ 开头：颜色掩码行，每个字符对应上方文字同一位置的颜色
      - . 或空格：默认颜色
      - 字母（如 R、G、C 等）：对应 ANSI 颜色

    示例：
      --[ ink ]--
      |Hello World
      ~WWWWW.GGGGG

    渲染效果：Hello 为亮白色，World 为亮绿色。

┌─ 8.3 ───────────────────────────────────────────────[ 框线绘制 ]─┐

    使用 Unicode Box Drawing 字符绘制终端风格边框。常用字符：

      ─ │ ┌ ┐ └ ┘ ├ ┤ ┬ ┴ ┼    框线
      ▲ ▼ ► ◄                    箭头
      ✓ ✗                        验证标记

    框线用于：
    - 表格边框（见 3.7 节）
    - 流程图连线（见 3.5 节）
    - 树形结构（INK KEY、PROOF CHAIN 等）

    注意：框线字符是全角宽度，在编辑器中需要等宽字体才能正确对齐。

──[ 9.0 ]──────────────────────────────────────────────────────[ SEO 优化要点 ]

    本站 SEO 功能包括：RSS Feed（/rss.xml）、robots.txt、以及页面 meta 标签。
    注意：sitemap 功能已移除，搜索引擎通过 robots.txt 和内部链接发现页面。

┌─ 9.1 ───────────────────────────────────────────────[ 标题与描述 ]─┐

    - title 字段应包含文章核心关键词，长度控制在 60 字符以内
    - 文章开头的第一段（摘要）会被搜索引擎用作描述（description）
    - 摘要段落应清晰概括文章内容，包含主要关键词
    - 章节标题（──[ N ]──[ Title ]）中的 Title 应使用英文关键词

┌─ 9.2 ───────────────────────────────────────[ 图片 Alt 文本 ]─┐

    - 每张图片必须有描述性的 alt 文本
    - alt 文本应简洁描述图片内容（10-20 字为宜）
    - 包含关键词但不要堆砌
    - 纯装饰性图片仍需填写 alt 文本

┌─ 9.3 ───────────────────────────────────────────────[ 链接规范 ]─┐

    - 链接文字应描述目标内容，避免 "点击这里"、"查看" 等无意义文字
    - 外部链接使用完整 URL（含 https://）
    - 引用来源时，在文章末尾的 References 章节集中列出

    正确：

    错误：
