# .phile 语法规则指南

`.phile` 文件本质是 **Markdown + YAML frontmatter**，额外支持 ANSI 终端风格的彩色标记和框线绘制。本文档分为「语法篇」和「结构篇」两部分。

---

# 语法篇

## 1. Frontmatter（文件头元数据）

每个 `.phile` 文件必须以 YAML frontmatter 开头，夹在 `---` 之间。

```yaml
---
title: "文章标题" # 必需，显示在文章顶部
date: 2026-06-17 # 必需，日期（YYYY-MM-DD）
author: "作者名" # 必需，作者名称
lang: zh # 可选，语言代码（en/zh），默认 en
order: 0 # 可选，排序权重（数字越小越靠前）
redacted: false # 可选，是否隐藏该文章，默认 false
---
```

---

## 2. ANSI 颜色标记

格式：`\#[颜色代码|文字内容]`

### 2.1 标准颜色（小写字母）

| 标记 | 颜色   | 终端效果  |
| ---- | ------ | --------------------------------------- |
| `#[k | text]` | 黑色      | 在深色背景上几乎不可见   |
| `#[r | text]` | 红色      | 错误、警告、负面信息     |
| `#[g | text]` | 绿色      | 成功、验证通过、正确路径 |
| `#[y | text]` | 黄色      | 注意、偏移量、边界值     |
| `#[b | text]` | 蓝色      | 次要信息                 |
| `#[m | text]` | 品红/紫色 | 特殊标记                 |
| `#[c | text]` | 青色      | 格式标记、容器节点       |
| `#[w | text]` | 白色      | 默认文本色               |

### 2.2 亮色（大写字母）

| 标记 | 颜色   | 终端效果   |
| ---- | ------ | ------------------------------------ |
| `#[K | text]` | 亮黑（灰） | 注释、次要文本       |
| `#[R | text]` | 亮红       | 高危警告、错误路径   |
| `#[G | text]` | 亮绿       | 关键数据、验证通过   |
| `#[Y | text]` | 亮黄       | 高亮提示、重要信息   |
| `#[B | text]` | 亮蓝       | 强调信息             |
| `#[M | text]` | 亮品红     | 特殊高亮             |
| `#[C | text]` | 亮青       | 链接、标题、关键标记 |
| `#[W | text]` | 亮白       | 粗体强调、主标题     |

### 2.3 颜色别名

除了单字母代码，也支持完整颜色名和缩写：

| 完整名称          | 缩写   | 等效          |
| ----------------- | ------ | ------------- |
| `#[red            | text]` | —             | `#[r   | text]` |
| `#[green          | text]` | —             | `#[g   | text]` |
| `#[yellow         | text]` | —             | `#[y   | text]` |
| `#[blue           | text]` | —             | `#[b   | text]` |
| `#[magenta        | text]` | —             | `#[m   | text]` |
| `#[cyan           | text]` | —             | `#[c   | text]` |
| `#[white          | text]` | —             | `#[w   | text]` |
| `#[bright-red     | text]` | `#[br-red     | text]` | `#[R   | text]` |
| `#[bright-green   | text]` | `#[br-green   | text]` | `#[G   | text]` |
| `#[bright-yellow  | text]` | `#[br-yellow  | text]` | `#[Y   | text]` |
| `#[bright-blue    | text]` | `#[br-blue    | text]` | `#[B   | text]` |
| `#[bright-magenta | text]` | `#[br-magenta | text]` | `#[M   | text]` |
| `#[bright-cyan    | text]` | `#[br-cyan    | text]` | `#[C   | text]` |
| `#[bright-white   | text]` | `#[br-white   | text]` | `#[W   | text]` |

### 2.4 嵌套与转义

ANSI 标记**不支持嵌套**。`\#[C|outer \#[R|inner]]` 中的 `inner` 不会被标红，`\#[R|` 会被当作普通文字显示。

需要显示 `#`、`[`、`]`、`|`、`\` 等特殊字符时，使用反斜杠转义：

```
\[   —— 显示为 [
\]   —— 显示为 ]
\|   —— 显示为 |
\\   —— 显示为 \
```

## 3. 框线绘制字符

使用 Unicode Box Drawing 字符绘制终端风格边框。

### 3.1 基本字符

| 字符 | Unicode | 名称    | 用途               |
| ---- | ------- | ------- | ------------------ |
| `─`  | U+2500  | 水平线  | 边框水平线、分隔线 |
| `│`  | U+2502  | 竖直线  | 流程图、树形结构   |
| `┌`  | U+250C  | 左上角  | 边框起始           |
| `┐`  | U+2510  | 右上角  | 边框结束           |
| `└`  | U+2514  | 左下角  | 边框起始           |
| `┘`  | U+2518  | 右下角  | 边框结束           |
| `├`  | U+251C  | 左 T 形 | 树形分支           |
| `┤`  | U+2524  | 右 T 形 | 树形收尾           |
| `┬`  | U+252C  | 上 T 形 | 表格分隔           |
| `┴`  | U+2534  | 下 T 形 | 表格分隔           |
| `┼`  | U+253C  | 十字    | 表格交叉           |

### 3.2 其他常用字符

| 字符 | Unicode | 用途     |
| ---- | ------- | -------- |
| `►`  | U+25BA  | 流向箭头 |
| `▼`  | U+25BC  | 向下箭头 |
| `✓`  | U+2713  | 验证通过 |
| `✗`  | U+2717  | 验证失败 |

## 4. INK 块（渐变文字）

INK 块用于实现逐字不同颜色的渐变效果。以 `--[ ink ]--` 开头，空行结束。

### 4.1 语法

```
--[ ink ]--
|文字行内容
~颜色掩码
```

- `|` 开头：要显示的文字行，`|` 不会被渲染
- `~` 开头：颜色掩码行，每个字符对应上方文字同一位置的颜色
  - `.` 或 ` `：默认颜色
  - 字母（如 `R`、`G`、`C` 等）：对应 ANSI 颜色

### 4.2 示例

```
--[ ink ]--
|Hello World
~WWWWW.GGGGG
```

渲染效果：`Hello` 为亮白色，`World` 为亮绿色。

## 5. Markdown 自动转换

项目会自动将标准 Markdown 语法转换为 ANSI 终端风格，无需手动编写 ANSI 标记。

### 5.1 标题

```
# 一级标题
## 二级标题
```

### 5.2 行内格式

| 语法          | ANSI 效果    |
| ------------- | ------------ |
| `**粗体**`    | 亮白色（`#[W | text]`） |
| `*斜体*`      | 青色（`#[c   | text]`） |
| `` `代码` ``  | 亮绿色（`#[G | text]`） |
| `[链接](url)` | 亮青色（`#[C | text]`） |

### 5.3 引用块

```
> 引用文字
```

渲染为带 `>` 前缀的 ANSI 引用块。

### 5.4 代码块（仅上下边框，无侧边框）

````
```bash
echo "hello"
echo "world"
```
````

渲染为：

```
┌─ bash ───────────────────────────────────────────────────────────┐
echo "hello"
echo "world"
└──────────────────────────────────────────────────────────────────┘
```

### 5.5 容器（仅上下边框，无侧边框）

```
:::important
这是重要提示
:::
```

渲染为：

```
┌─ IMPORTANT ──────────────────────────────────────────────────────┐
这是重要提示
└──────────────────────────────────────────────────────────────────┘
```

支持的容器类型：

| 标记           | 边框颜色 | 用途     |
| -------------- | -------- | -------- |
| `:::important` | 亮黄色   | 重要提示 |
| `:::note`      | 青色     | 备注信息 |
| `:::tip`       | 绿色     | 技巧提示 |
| `:::warning`   | 红色     | 警告信息 |

### 5.6 制表符处理

源代码中的 `\t` 制表符（Tab）会自动展开为 4 个空格，确保代码块和容器内文本对齐。

# 结构篇

## 6. 文章整体布局

一篇典型的 `.phile` 文章从上到下包含以下结构：

```
[Frontmatter 元数据]

[封面标题区]
──[ 1.0 ]──...──[ 标题 ]

[文章正文]
... 内容 ...

[章节分隔]
──[ 2.0 ]──...──[ 标题 ]

[文章正文]
... 内容 ...

[可能更多章节...]

[图表/流程图]
[代码块]
[容器（提示框）]
[INK 块]
```

---

## 7. 章节标题

### 格式

```
──[ 编号 ]──...──[ 标题文字 ]
```

编号和标题文字之间的 `─` 数量由渲染器自动计算，填满整行。

### 约定

| 编号  | 含义      |
| ----- | --------- |
| `0.0` | 摘要/概述 |
| `1.0` | 第一章节  |
| `2.0` | 第二章节  |
| `3.0` | 第三章节  |
| `...` | 依此类推  |

每个章节标题上下应有空行，形成视觉分隔。

---

## 8. 图例说明

在文章开头（通常紧随 `0.0 Summary` 之后），可使用树形结构列出颜色图例：

```
  INK KEY:
  ├─ #[C|C]  format markers & container nodes
  ├─ #[G|G]  validated / recovered data
  ├─ #[Y|Y]  offsets, sizes, boundary values
  └─ #[R|R]  wrong paths, hazards, negative controls
```

### 模式

- 首行：`  LABEL NAME:`（两个空格缩进）
- 分支：` ├─` 或 ` └─`（两个空格 + 树形字符 + 空格）
- 每个分支包含一个对应的 ANSI 颜色标记和说明文字

---

## 9. 流程图

使用框线字符和箭头绘制数据流向图。

### 示例

```
  DECRYPTION FLOW:
  vendor ZIP
      │
      ▼
  update BIN
      ├─ 0x00000000  #[C|vendor header]
      ├─ 0x00000200  #[G|encrpted_img]
      └─ 0x00000400  #[Y|trailer]
```

### 模式

- 首行：`  LABEL:`（两个空格缩进 + 标签 + 冒号）
- 节点：直接写文字（缩进 6 个空格）
- 连线：`│`（缩进 6 个空格）
- 箭头：`▼`（缩进 6 个空格）
- 分支：`├─` 或 `└─`（缩进 6 个空格）
- 每个分支使用 ANSI 颜色标记突出关键信息

---

## 10. 证明链

用于展示验证步骤的有序列表。

### 示例

```
  PROOF CHAIN:
  ├─ [1] #[C|wrapper header]  ✓
  ├─ [2] #[G|positive control]  ✓
  └─ [3] #[R|negative control]  ✗
```

### 模式

- 首行：`  LABEL:`（两个空格缩进）
- 每步：`  ├─ [n] ANSI_TEXT  [✓/✗]`
- 最后一步：使用 `└─`
- 使用 ANSI 颜色标记关键步骤名称
- 使用 `✓` 表示通过，`✗` 表示失败

---

## 11. 表格结构

使用框线字符绘制表格。

### 示例

```
  ┌──────────────────┬──────────────┬──────────────────┐
  │ Field            │ Offset       │ Value            │
  ├──────────────────┼──────────────┼──────────────────┤
  │ magic            │ 0x00000000   │ #[G|ENCR]        │
  │ version          │ 0x00000004   │ #[C|0x00000001]  │
  │ size             │ 0x00000008   │ #[Y|0x00001000]  │
  └──────────────────┴──────────────┴──────────────────┘
```

### 模式

- 表格缩进 2 个空格
- 使用 `┌─┬─┐` / `├─┼─┤` / `└─┴─┘` 分别绘制顶、中、底边框
- 表头行使用 `│` 分隔列
- 数据行使用 `│` 分隔列，每列手动对齐（用空格填充）
- 关键数据使用 ANSI 颜色标记

## 12. 完整书写示例

````phile
---
title: "示例技术分析"
date: 2026-06-17
author: "AH"
lang: zh
order: 1
---

──[ 0.0 ]──────────────────────────────────────────────────[ Summary ]

这是一个 #[Y|重要] 的发现。我们使用了 #[G|AES-CBC] 加密算法。

  INK KEY:
  ├─ #[C|C]  format markers
  ├─ #[G|G]  validated data
  ├─ #[Y|Y]  offsets
  └─ #[R|R]  wrong paths

──[ 1.0 ]──────────────────────────────────────────────[ Introduction ]

文件结构如下：

  FILE STRUCTURE:
  vendor ZIP
      │
      ▼
  update BIN
      ├─ 0x00000000  #[C|vendor header]
      └─ 0x00000200  #[G|payload]

──[ 2.0 ]──────────────────────────────────────────────────[ Analysis ]

验证结果：

  PROOF CHAIN:
  ├─ [1] #[C|wrapper header]  ✓
  ├─ [2] #[G|positive control]  ✓
  └─ [3] #[R|negative control]  ✗

:::important
#[Y|请确保已备份数据] 再进行操作
:::

```bash
# 解密命令
openssl enc -d -aes-256-cbc -in payload.bin -out decrypted.bin
````

─[ 3.0 ]───────────────────────────────────────────────[ Conclusion ]

总结：[链接](https://example.com)

--[ ink ]--
|Gradient Example
~GGGGGGG.....RRRR

